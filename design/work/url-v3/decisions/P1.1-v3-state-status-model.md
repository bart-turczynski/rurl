# P1.1 — v3 state and status model

## Decision (PROPOSED)

> **Proposed, not binding.** This record is authored by the agent at
> `state: PROPOSED`. It presents the agent's proposed v3 state/status model for
> the owner to ratify, reject, or adjust. It becomes `ACCEPTED` only when the
> owner merges it (P0.1 §4, §Ratification); until then it settles nothing and
> no downstream artifact may treat it as authority.

This record settles the **P1 half** of contradiction **C-07** — the canonical
state/status *model* and the legacy-projection *principle*. It deliberately does
**not** settle the validation-verdict *semantics* or the migration *mechanics*,
which are the sibling record **P2.3**'s scope (see §Scope boundary).

### 1. Canonical state schema and the public/internal split

1.1 **One canonical record is the v3 truth.** v3 defines a single canonical URL
state record as the authoritative representation of a parsed URL. It is a
*superset* of both current representations: the **18** public parsed-output
fields (`.spu_result_fields`, `R/utils.R:215-241`) and the **20** Stage-A
internal cache fields (`.spu_stage_a_fields`, `R/utils.R:252-286`), plus the
structural *kinds* that today are neither public nor cached and are recovered by
a Stage-B reparse of `original_url` (`path_kind`, `authority_kind`, `query_kind`,
`fragment_kind`, host/authority forms; `R/parse-state.R:25-57`; S1-F1). No
canonical fact may be reconstructible only by reparsing discarded source
(S1-F1, S1-F11).

1.2 **Three-valued presence is canonical.** Every delimited component (authority,
host, userinfo/user/password, port, query, fragment, and path where applicable)
carries an explicit presence state drawn from **`absent`** (delimiter not
present), **`present-empty`** (delimiter present, payload empty), and
**`present-nonempty`**, held distinctly from its payload value. This ends the
current collapse where empty and absent host/query/fragment both project to a
single `NA` (S1-F1 `foo:///x?#`; S7-F3). Absence and present-emptiness MUST
remain distinguishable in the canonical record even where a compatibility
projection renders both as missing (S7-F3).

1.3 **Provenance is kept per component, not overloaded.** For each component the
canonical record keeps the distinct forms named by S1-F9: immutable source
slice, parsed semantic identity, standard-serialization value, and display
value (plus a cleaning projection where applicable). A single string field MUST
NOT stand for several of these at once. This record fixes that these forms are
*distinct*; their exact per-field value domains are instantiated by the §6
artifact-3 canonical-state contract at G3.

1.4 **Public/internal split — the principle.** The canonical record (internal
truth) is partitioned by role:

- **Public** = identity-bearing and presentation-facing fields — the components
  a caller reads or renders, and the stable identity keys. At the baseline these
  are exactly the 18 `.spu_result_fields`, retained as the v3 **public
  projection**.
- **Internal** = raw component slices and classifier flags that exist to drive
  parsing, caching, and derivation, not to be read as results — the Stage-A
  raw/flag fields (`raw_path`, `raw_query`, `raw_fragment`, `raw_user`,
  `raw_password`, `raw_port`, `host_is_ace`, `looks_like_protocol`,
  `original_has_allowed_scheme`, `is_scheme_relative`, `looks_like_host_port`,
  `scheme_less_userinfo`, `rfc3986_path_rootless`) and the structural kinds.

The **criterion** is role, not storage location: a field is public iff it is a
result a caller consumes; it is internal iff it exists only to produce results.
`is_ip_host` deliberately spans both (public field and Stage-A flag;
`R/utils.R:238`, `R/utils.R:255`) and stays public because it is a consumed
fact.

1.5 **The 18-field frame is a compatibility projection, not the v3 object.**
Per the owner standing mandate (standards-exactness / correctness beats
back-compat), the v2 18-column data frame is retained as a **compatibility
view** over the canonical record, not re-adopted as the v3 canonical shape.
Whether v3 *also* exposes a first-class URL object/class in addition to the
projection was **resolved at the 2026-07-23 ratification review (B3): a
first-class public URL object/class is deferred to a later minor.** v3.0 ships
the 18-field data frame + scalar list as the **sole public surface**, now framed
explicitly as a compatibility projection over the internal canonical record. The
canonical record remains the internal v3 truth; no new public class is frozen at
the 3.0 major boundary. (See §Owner ratification.)

### 2. The layered status model that resolves C-07

C-07 is that today's single `parse_status` column conflates three unrelated
concerns. Verified at the baseline in `.derive_parse_status_vec()`
(`R/parse-phases.R:2614-2688`): one enum column mixes (a) **syntax/parse
validity** (`error` when `!curl_ok`), (b) **policy/admission** (`error` when a
token looks like an unsupported scheme under `scheme_acceptance = "web"`;
`ok-ftp`; `ok-scheme-relative`; and, from `status-constants.R`,
`warning-userinfo`), and (c) **optional PSL-derived facts** (`warning-no-tld`,
`warning-invalid-tld`, `warning-public-suffix`, decided from the PSL-derived
`tld`/`domain` values).

v3 replaces the single column with a **structured, multi-layer verdict record**
with three independent layers, each with its own typed vocabulary:

- **Layer 1 — Syntax / parse verdict.** Did the input parse into a well-formed
  URL under the selected standard? This is the pure standard-parse outcome and
  depends on nothing outside the URL string and the standard. It is computed
  **first**.
- **Layer 2 — Policy / admission verdict.** The outcome of the input-acceptance
  axes (`scheme_policy`, `scheme_acceptance`; ADR 0010/0012) and related
  policy outcomes (scheme-relative handling, FTP-family treatment, userinfo
  policy). Policy is layered **over** a syntactically parsed URL; it is not
  syntax.
- **Layer 3 — Optional annotation facts.** PSL registrability facts
  (public-suffix / registrable-domain knowledge) are **optional annotations**,
  not parse validity (S7 §3, S7-F3, S7 owner-decision 1). They MUST be typed
  annotation results carrying an explicit state (at least the S7-F3 set:
  `not-requested`, `not-applicable`, `known`, `unknown`, `invalid-input`,
  `dependency-error`) plus value/provenance — never a bare `NA` standing for
  several states.

**Load-bearing invariant (the resolution of C-07).** Layer 3 MUST NOT change
Layer 1 or Layer 2. The syntax and policy verdicts are decided **before**
optional annotation execution and remain invariant under annotation selection,
cache warmth, PSL section/list/engine version, and dependency availability
(S7-F3). A skipped or unknown PSL lookup can never alter the apparent
parse verdict — which is exactly the defect C-07 names.

This record fixes the *layers* and this *independence invariant*. The exact enum
membership of each layer, which policy outcomes are fatal versus warning, and how
annotation states resolve are P2.3 (§Scope boundary).

### 3. Result type and failure / recovery shape

3.1 **Vector recovery is row-local.** Vectorized operations recover per element:
a failed element yields a row whose canonical record holds the typed
invalid/absent state (Layer-1 verdict = unparseable), and the surrounding call
completes — it never aborts because one element failed. This preserves today's
`safe_parse_urls()` row-local error-row behavior
(`R/parse.R:585-625`; public-surface-inventory `safe_parse_urls` row).

3.2 **A failed element is a first-class typed state, not an absence.** The v3
canonical result of a failed element is a **typed invalid canonical record**
that still carries `original_url` provenance and the layered verdicts — because
the canonical schema must be able to *represent* the invalid state (S1-F1
"invalid-state representation"), not signal it only by `NULL`/`NA`. The legacy
scalar `safe_parse_url()` `NULL`-on-failure and the legacy one-row error-row are
retained as **compatibility projections** over that typed invalid record.

3.3 **Recycling is preserved.** Scalar behavior dials recycle against the `url`
vector as today; the canonical result length follows the input length.

3.4 **Interface, not the full matrix.** The governing principle is: *row-local
recovery for parse/validation failure; call-level errors reserved for contract
violations (e.g. an unusable argument), not for bad data.* The exhaustive
scalar/vector cell matrix (S1-F3: `NULL`/`character(0)`/factor/list-element
length 0/1/>1/mixed vectors, names, cold/warm cache) is the S1 scalar/vector
contract artifact and is not enumerated here; this record fixes the recovery
shape and result-type stance the matrix must satisfy.

### 4. Legacy `parse_status` preserved as an explicit compatibility projection

The current single-column `parse_status` (values `ok`, `ok-ftp`,
`ok-scheme-relative`, `error`, `warning-no-tld`, `warning-invalid-tld`,
`warning-public-suffix`, `warning-userinfo`; `R/status-constants.R`), derived by
`.derive_parse_status_vec()` from PSL-derived `tld`/`domain` warnings (§10;
`R/parse-phases.R:2614-2688`), is **retained** — but only as an explicit,
documented **compatibility projection** over the layered model, never as the v3
truth.

**Projection principle.** `parse_status = π(layer-1 syntax, layer-2 policy,
layer-3 annotation)` where π is a **total, pure** collapse function that reads
*only* the three layers and introduces no fact not already present in them. Two
consequences are normative:

- The projection is the **only** place where an optional PSL fact is allowed to
  influence a status-shaped output. No v3-canonical consumer reads `parse_status`
  to decide syntax validity; syntax validity is read from Layer 1.
- Because π is total over the layers, the legacy column is reproducible for
  back-compat without re-conflating the layers in the canonical model — the
  conflation lives only in the lossy view, by design.

The exact collapse table (precedence when several layers are non-nominal, e.g.
error-over-warning, which PSL fact wins) and the deprecation/migration steps for
`parse_status` and `get_parse_status()` are **P2.3** (§Scope boundary).

### Scope boundary — deferred to P2.3 (RCON-04, also C-07)

This record defines the **model**. Sibling record **P2.3** owns, and this record
does **not** settle:

- **Validation-verdict semantics** — the concrete enum membership of each layer;
  which policy outcomes are **fatal** versus **warning**; the annotation-state
  resolution rules (how the S7-F3 typed states are decided and surfaced);
  reference/resolution behavior (RCON-04).
- **Legacy-status migration mechanics** — the exact π collapse table, the
  deprecation schedule, and the accessor/column changes for `parse_status`,
  `get_parse_status()`, and `get_url_diagnostics()`.

**Interface P2.3 MUST honor:** (i) three independently addressable layers
exist — syntax, policy, optional annotation; (ii) Layer 3 cannot alter Layer 1
or Layer 2, and syntax/policy are computed before annotation (§2 invariant);
(iii) optional annotations are typed results, not bare `NA`; (iv) legacy
`parse_status` is a total, pure projection of the three layers and is the sole
locus where PSL facts reach a status output (§4).
